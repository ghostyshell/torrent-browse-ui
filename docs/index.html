Everything you need

One interface for searching, streaming, bookmarking, and managing your torrent media.

Multi-indexer search

Search PirateBay, YTS, 1337x, Nyaasi, LimeTorrent, and more from a single interface. Switch indexers with one click.

Real-Debrid streaming

Add a magnet link, get a direct stream URL in seconds. Your API key is stored server-side — never in the browser.

Favourites

Save torrents to your favourites list. Stored on the backend so your list is available across sessions and devices.

Cached stream links

Cache resolved stream URLs with automatic expiry detection. One-click refresh all expired magnet links.

Cover image management

Automatically fetch and store cover images via the backend image proxy. Search Google Images or paste any URL manually.

Google Cast support

Cast any resolved stream URL directly to a Chromecast. Compatible with the default Google media receiver — no custom app required.

How it works

From search query to stream URL in four steps.

  1. 1

    Configure your backend

    Point REACT_APP_API_URL at a compatible torrent-search / streaming backend API. The frontend delegates all indexer scraping, database operations, and Real-Debrid proxying to it.

  2. 2

    Sign in with Google

    The backend handles Google OAuth. After login, a bearer token is stored in sessionStorage and attached to every API request. Your Real-Debrid API key (if provided) is stored server-side only.

  3. 3

    Search and browse

    Type a query, select an indexer, apply quality or seeder filters, and paginate results. All search parameters live in the URL — results pages are bookmarkable and shareable.

  4. 4

    Stream or save

    Click a result to view details and the magnet link. Add to favourites or stored links, or hit Stream to resolve a Real-Debrid direct URL and play in the built-in Video.js player — or cast to your TV via Chromecast.

Configuration highlights

All settings are injected at build time — no config files needed at runtime.

REACT_APP_API_URL

Backend API base URL. Required. Set via --build-arg in Docker or as a CI environment variable.

REACT_APP_USE_BACKEND_FIRST

Defaults to true. Reads favourites and links from the backend first; falls back to localStorage on failure.

REACT_APP_SHOW_CONSOLE_LOGS

Set to OFF to enable browser console logs (counterintuitive: ON = suppressed, OFF = visible).

REACT_APP_CAST_RECEIVER_APP_ID

Chromecast receiver app ID. Defaults to Google's Default Media Receiver — works without a custom receiver app.

Quickstart

Up and running in under five minutes.

1

Clone the repository

git clone https://github.com/akshatsinghkaushik/stream-frontend.git
cd stream-frontend
2

Build the Docker image

docker build \
  --build-arg REACT_APP_API_URL=https://your-backend.example.com \
  --build-arg REACT_APP_BACKEND_URL=https://your-backend.example.com \
  -t stream-frontend .
3

Run the container

docker run -p 3000:80 stream-frontend
# Open http://localhost:3000
4

Or run locally

cp .env.example .env.local
# Set REACT_APP_API_URL in .env.local

npm ci
npm start       # http://localhost:3000

Frequently asked questions

What backend does this UI require?
stream-frontend requires a compatible torrent-search and streaming backend API, configured via the REACT_APP_API_URL environment variable at build time. The frontend project does not include or operate a backend service.
Is a Real-Debrid account required?
No. You can search and browse torrents without Real-Debrid. The Real-Debrid integration is optional and enables resolving magnet links into direct stream URLs. Without it, you can still copy magnet links and open them in a torrent client.
Where is my Real-Debrid API key stored?
Server-side only. It is saved to the backend database when you submit it in Account Settings and is never written to localStorage, sessionStorage, or any browser cookie. See the privacy policy for full details.
Does it work in private / incognito mode?
Yes. When the app detects a private browsing context (where localStorage is unavailable or severely quota-limited), it automatically switches to backend-only storage mode. All data is fetched from and saved to the backend, with no browser storage used. A notification banner is shown when this mode is active.
Can I use this on mobile?
Yes. The standard build is responsive and works in mobile browsers. There is also a dedicated Android build target (npm run build:android) that activates touch optimisations, larger tap targets, and a condensed layout — suitable for loading in a WebView.
