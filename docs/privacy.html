stream-frontend Privacy Policy

Privacy Policy

Last updated: June 2026

Overview

stream-frontend is an open-source web application. This policy explains what information is stored in your browser, what is transmitted to the backend API you configure, and how your credentials are handled. There is no analytics, no advertising, and no data sale.

What is Stored Locally in Your Browser

Session Token (sessionStorage)

After you log in, a short-lived bearer token is stored in sessionStorage. It is cleared automatically when you close the browser tab or call logout. It is never written to localStorage or cookies on the client side.

Cached Links Snapshot (localStorage)

When backend-first storage is active, a local copy of your stored links (magnet URLs, resolved stream URLs, titles, timestamps) is written to localStorage as a performance cache. The authoritative copy lives on your backend. You can clear this at any time by clearing your browser's site data.

Storage Configuration (localStorage)

The app's storage strategy settings (backend-first, localStorage fallback, backend-only mode) are persisted in localStorage under the key torrent_storage_config.

Cover Image URLs (localStorage, legacy)

In older versions, cover image URLs were cached in localStorage. Current versions store them on the backend instead. A one-time migration prompt offers to move any remaining local cover URLs to the backend.

In-Memory Caches (runtime only)

Torrent detail pages, magnet links, and resolved stream URLs are kept in memory during your session to avoid redundant API calls. These are discarded when the page is closed.

What is Sent to the Backend

Every data operation — favourites, stored links, cover images, torrent searches — is sent to the backend API at the URL you configure in REACT_APP_API_URL. You control where this backend runs. The frontend project does not ship with or operate any shared backend.

  • Your Google account email and profile information (used for authentication and the allow-list check).
  • Search queries you type and the torrent sites you select.
  • Magnet links and direct stream URLs you add to your stored links.
  • Cover image URLs you associate with torrents.
  • Requests to the Real-Debrid proxy endpoint (your API key is stored server-side and never sent from the browser).

Real-Debrid API Key

Your Real-Debrid API key is stored server-side only — it is saved to your backend's database when you submit it in Account Settings and is never written to localStorage, sessionStorage, or any browser cookie. All Real-Debrid API calls are proxied through the backend; the browser only receives the resolved stream URL.

You can remove your Real-Debrid key at any time from the Account Settings page, which instructs the backend to delete it from its database.

Incognito / Private Browsing

When the app detects that it is running in a private browsing context (where localStorage is unavailable or severely quota-limited), it automatically switches to backend-only mode. In this mode, no data is written to the browser at all — everything is stored on the backend and retrieved on each page load. A notification banner is shown when this mode is active.

Contact

This is an open-source project. For questions about data handling, open an issue on the GitHub repository.
